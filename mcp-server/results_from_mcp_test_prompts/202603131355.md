Issues Found

1. Quickstart creates empty-scope credentials — scoped authenticate calls fail unless you manually
   create_credential with scopes first. Consider adding a scopes parameter to quickstart.
2. No agent visibility isolation — list_agents and get_agent return all agents regardless of caller's
   trust hierarchy. Any authenticated agent can enumerate every other agent in the system. This is a
   security concern for multi-tenant deployments.

Overall: 21/23 PASS, 1 PARTIAL, 1 FAIL. Core auth flows (token lifecycle, credential management,
delegation, revocation cascading) are solid. The two gaps are in scope handling on quickstart and
trust-hierarchy visibility filtering
